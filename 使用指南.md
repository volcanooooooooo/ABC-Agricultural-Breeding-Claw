## 系统简介

ABC (Agricultural Breeding Claw) 是一个面向育种研究人员的智能分析助手，通过自然语言交互完成复杂的生物信息学分析任务。

### 核心特性

- **自然语言交互**: 用中文或英文描述分析需求，系统自动理解并执行
- **双轨分析**: 同时使用传统统计工具和 AI 大模型进行分析对比
- **多种分析工具**: 差异表达分析、富集分析、BLAST 序列比对
- **可视化结果**: 火山图、富集气泡图、KEGG 通路图
- **数据管理**: 支持上传自定义数据集，自动识别分组

## 快速开始

### 1. 启动系统

在项目根目录执行：

```bash
./start.sh
```

系统将自动启动前后端服务：
- **前端界面**: http://localhost:3003
- **后端 API**: http://localhost:8003
- **API 文档**: http://localhost:8003/docs

### 2. 停止系统

```bash
./stop.sh
```

### 3. 首次使用

1. 打开浏览器访问 http://localhost:3003
2. 点击右上角"登录"按钮（可选，游客模式可直接使用）
3. 进入设置页面配置 LLM API（推荐使用千问 API）

## 功能模块

### 1. 对话分析页面（主页）

这是系统的核心交互界面，支持自然语言对话和多种分析任务。

#### 会话管理
- **新建会话**: 点击左侧"新建对话"按钮
- **切换会话**: 点击左侧会话列表中的任意会话
- **删除会话**: 鼠标悬停在会话上，点击删除图标
- **会话持久化**: 所有会话自动保存到本地

#### 快捷命令
在输入框中输入以下命令：
- `/tools` - 显示所有可用的分析工具
- `/datasets` - 显示所有可用的数据集

#### 支持的分析类型

**差异表达分析**
```
分析 WT 和 osbzip23 的差异表达基因
比较对照组和处理组的差异
/analyze --control WT --treatment osbzip23
```

**富集分析**
```
对这些基因做富集分析
帮我分析基因的功能富集
```

**BLAST 序列比对**
```
帮我做 BLAST 分析
比对这个序列
```

**基因查询**
```
查询基因 LOC_Os01g01010
这个基因的功能是什么
```

### 2. 分析页面

提供结构化的分析工作流程。

#### 数据集管理标签

**查看数据集**
- 显示所有已上传的数据集
- 查看数据集元数据（基因数、样本数、创建时间）
- 删除不需要的数据集

**上传数据集**
1. 点击"上传数据集"按钮
2. 填写数据集信息：
   - 数据集名称
   - 描述信息
   - 对照组名称（如 "WT"）
   - 处理组名称（如 "osbzip23"）
   - 对照组样本列表（逗号分隔）
   - 处理组样本列表（逗号分隔）
3. 选择文件（支持 CSV、Excel 格式）
4. 点击"上传"

**数据格式要求**
```
Gene_ID    Sample1    Sample2    Sample3    ...
Gene1      100        150        120        ...
Gene2      200        180        210        ...
```

#### 双轨分析标签

**执行分析**
1. 选择数据集
2. 输入对照组名称（如 "WT"）
3. 输入处理组名称（如 "osbzip23"）
4. 点击"开始分析"

**查看结果**
- **进度面板**: 实时显示分析进度
- **工具分析结果**: 传统统计方法的结果
  - 差异基因列表
  - 火山图可视化
  - 统计参数（p-value、log2FC）
- **LLM 分析结果**: AI 大模型的解读
  - 生物学意义解释
  - 关键基因识别
  - 功能推断

**提供反馈**
- 每个分析结果下方有反馈按钮
- 点击"有帮助"或"无帮助"
- 可选填写详细反馈意见

## 使用教程

### 教程 1: 差异表达分析

**场景**: 比较野生型（WT）和突变体（osbzip23）的基因表达差异

**步骤**:

1. **使用内置数据集**
   ```
   分析 WT 和 osbzip23 的差异表达基因
   ```

2. **系统自动执行**:
   - 选择数据集 GSE242459
   - 识别对照组和处理组样本
   - 执行双轨分析（统计工具 + AI 模型）

3. **查看结果**:
   - **统计结果**: 显著差异基因数量、火山图
   - **AI 解读**: 生物学意义、关键通路、功能推断

4. **下载结果**:
   - 点击"下载结果"按钮
   - 获取差异基因列表（CSV 格式）

### 教程 2: 上传自定义数据集

**场景**: 分析自己的 RNA-seq 数据

**步骤**:

1. **准备数据文件**
   - 格式: CSV 或 Excel
   - 第一列: 基因 ID
   - 其他列: 样本表达量
   - 列名建议包含分组信息（如 WT_rep1, Treatment_rep1）

2. **上传数据集**
   - 进入"分析页面" → "数据集管理"
   - 点击"上传数据集"
   - 填写表单：
     ```
     数据集名称: 我的水稻实验
     描述: 2024年春季田间试验数据
     对照组: WT
     处理组: Mutant
     对照组样本: WT_rep1,WT_rep2,WT_rep3
     处理组样本: Mutant_rep1,Mutant_rep2,Mutant_rep3
     ```
   - 选择文件并上传

3. **执行分析**
   - 切换到"双轨分析"标签
   - 选择刚上传的数据集
   - 输入分组名称
   - 开始分析

### 教程 3: 富集分析

**场景**: 对差异基因进行功能富集分析

**步骤**:

1. **获取差异基因列表**
   - 先执行差异表达分析
   - 记录显著差异基因

2. **执行富集分析**
   ```
   对这些差异基因做富集分析
   ```
   或上传基因列表文件

3. **查看结果**:
   - **GO 富集**: 生物学过程、分子功能、细胞组分
   - **KEGG 富集**: 代谢通路、信号通路
   - **气泡图**: 可视化富集结果
   - **通路图**: KEGG 通路详细图谱

4. **解读结果**:
   - 查看 AI 对富集结果的生物学解释
   - 识别关键功能类别和通路

### 教程 4: BLAST 序列比对

**场景**: 比对未知序列找同源基因

**步骤**:

1. **准备序列文件**
   - 格式: FASTA
   - 可包含单条或多条序列

2. **上传并分析**
   ```
   帮我做 BLAST 分析
   ```
   - 系统提示上传文件
   - 选择 FASTA 文件
   - 自动执行 BLAST

3. **选择数据库**
   - 系统显示可用的 BLAST 数据库
   - 选择目标数据库（如 MH63 基因组）

4. **查看结果**:
   - 比对命中列表
   - E-value、比对长度、相似度
   - 基因注释信息

### 教程 5: 使用文件上传功能

**场景**: 在对话中上传数据文件

**步骤**:

1. **点击上传按钮**
   - 输入框左侧的"📎"图标

2. **选择文件类型**:
   - **表达矩阵**: CSV/Excel 格式的基因表达数据
   - **FASTA 序列**: 用于 BLAST 分析
   - **基因列表**: 用于富集分析

3. **自动识别分组**
   - 系统自动从列名推断分组
   - 如 "WT_rep1" → 对照组 "WT"
   - 如 "Treatment_rep1" → 处理组 "Treatment"

4. **确认或修改分组**
   - 系统显示识别结果
   - 可手动调整分组分配

5. **执行分析**
   - 文件上传后自动触发相应分析

## 常见问题

### Q1: 如何获取千问 API Key？

**答**: 
1. 访问阿里云百炼平台: https://bailian.console.aliyun.com/
2. 注册并登录账号
3. 进入"API-KEY 管理"
4. 创建新的 API Key
5. 复制 Key 并粘贴到系统设置中

### Q2: 分析失败怎么办？

**答**:
1. **检查 API 配置**: 确保 API Key 正确且有余额
2. **检查数据格式**: 确保数据文件格式正确
3. **查看日志**: 
   - 后端日志: `logs/backend.log`
   - 前端日志: 浏览器控制台（F12）
4. **重试分析**: 有时网络问题会导致失败，可重试

### Q3: 支持哪些物种的数据？

**答**:
- 系统主要针对水稻（Rice）优化
- 理论上支持任何物种的基因表达数据
- BLAST 数据库目前包含：
  - MH63 水稻基因组
  - 可自行添加其他物种数据库

### Q4: 数据集大小有限制吗？

**答**:
- 单个文件建议不超过 100MB
- 基因数量建议不超过 50,000
- 样本数量建议不超过 1,000
- 超大数据集可能导致分析变慢

### Q5: 如何解读双轨分析结果？

**答**:
- **工具分析**: 基于统计学方法，结果客观可靠
- **LLM 分析**: 基于 AI 模型，提供生物学解释
- **对比使用**: 
  - 工具结果用于确定差异基因
  - LLM 结果用于理解生物学意义
  - 两者结合可获得更全面的认识

### Q6: 分析参数如何设置？

**答**:
- **P-value 阈值**: 默认 0.05
  - 更严格: 0.01（减少假阳性）
  - 更宽松: 0.1（增加灵敏度）
- **Log2FC 阈值**: 默认 1.0（2 倍差异）
  - 更严格: 2.0（4 倍差异）
  - 更宽松: 0.5（1.4 倍差异）

### Q7: 如何导出分析结果？

**答**:
1. **差异基因列表**: 点击"下载结果"按钮，获取 CSV 文件
2. **可视化图表**: 右键点击图表 → "图片另存为"
3. **对话记录**: 复制对话内容到文本编辑器
4. **完整报告**: 未来版本将支持一键导出 PDF 报告

### Q8: 系统安全吗？数据会泄露吗？

**答**:
- **本地部署**: 系统运行在本地服务器，数据不上传到云端
- **API 调用**: 仅在调用 LLM 时发送必要的分析参数
- **数据存储**: 所有数据集存储在本地 `backend/data/` 目录
- **建议**: 敏感数据请在内网环境使用

### Q9: 如何更新系统？

**答**:
```bash
# 停止服务
./stop.sh

# 拉取最新代码
git pull

# 更新依赖
cd backend && pip install -r requirements.txt
cd .. && npm install

# 重启服务
./start.sh
```

### Q10: 遇到问题如何获取帮助？

**答**:
1. **查看文档**: 阅读 `CLAUDE.md` 和本使用指南
2. **查看日志**: 检查 `logs/` 目录下的日志文件
3. **提交 Issue**: 在项目 GitHub 仓库提交问题
4. **联系开发者**: 通过项目维护者邮箱联系
