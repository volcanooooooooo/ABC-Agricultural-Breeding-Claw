1. 底层架构：基于图数据库的本体知识图谱
系统的底层数据不应仅仅是孤立的表格，而应该使用图数据库（如Neo4j）构建一张相互关联的本体网络
。
节点设计（Nodes）：包括数据集（Dataset）、样本（Sample，含分组属性）、基因（Gene）、表达量测量值（Measurement）、流程步骤（ProcessStep）、工具（Tool）、分析结果（Result）、结论（Conclusion）以及用户反馈（Feedback）等实体类型
。
关系设计（Edges）：节点之间通过定义好的语义关系相连，例如：数据集contains样本，样本has_measurement表达量，流程depends_on前置流程，流程uses_tool特定工具，结果supports结论等
。这种设计能够支持复杂的关联查询和分析推理。
2. 核心机制：双轨并行的分析工作流（Pipeline）
系统的分析执行依靠定义好的配置文件（如 pipeline_definition.json）进行驱动，将分析任务拆分为标准化步骤（如数据预处理、差异表达分析、功能富集分析、可视化报告生成）
。 对于每一个步骤，系统需支持双轨执行（Dual-track）
：
传统工具轨：通过预设的命令行调用标准的生信统计工具（如使用 fastp+DESeq2 进行预处理和差异分析，使用 clusterProfiler 进行富集分析）以保证严谨性
。
大模型（LLM）轨：基于提取出的数据摘要和特定的 Prompt 模板，调用大模型（如 DeepSeek 或 OpenClaw）进行推理和分析
。
3. 前端交互与数据流：全透明的对话式界面
前端应设计为一个支持流式输出和可视化卡片嵌入的网页聊天界面
。
意图解析与进度推送：用户输入自然语言请求后（如“做差异表达分析并双轨对比”），系统自动解析意图、匹配底层数据集（如寻找包含特定样本数的数据集），并通过流式文本向用户实时推送“双轨”各自的执行进度
。
结果对比卡片：分析完成后，系统生成可视化的对比卡片（JSON结构驱动）。卡片直观展示工具轨与大模型轨的分析结论对比（例如：共同检出的显著基因是谁、哪些基因是工具单独检出而大模型漏检的）
。
4. 进化机制：反馈闭环与知识沉淀
这是 ABC 系统实现”越用越聪明”的关键设计。
评价收集：在结果对比卡片下方，必须嵌入交互组件（点赞、点踩按钮及评论输入框），引导用户对双轨各自的表现进行评价
。
动态更新本体：用户的评价（如“大模型漏掉了Gene7，这个基因很重要”）会被系统捕获，转化为结构化数据写入底层的图数据库
。系统可以据此调整大模型流程节点的置信度（Confidence）和权重（Weight）
，或者直接在特定的基因节点（如 Gene7）上增加警告标签
。
经验复用：当用户下一次查询相关基因或发起类似分析时，系统可以结合之前的反馈记录（例如提示“有用户反馈大模型易漏检该下调基因”），辅助科研决策
。
