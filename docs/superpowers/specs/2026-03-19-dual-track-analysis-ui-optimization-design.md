# 双轨差异分析交互优化设计

**日期**: 2026-03-19
**状态**: 已确认

## 概述

根据用户与 ABC 系统交互数据流文档，对前端交互进行 P0 优先级优化，包含三个核心功能：

1. 进度步骤流展示
2. 追问卡片
3. 基因详情弹窗

---

## 1. 进度步骤流展示

### 目标

将分析过程中的进度以对话式消息流展示，而非单一进度条。

### 实现方案

当 SSE 推送进度更新时，**追加新消息**到会话中，而非更新现有进度条。

### 消息格式

```
[系统] 正在检索数据集...
[系统] 找到表达矩阵数据集（ID: expr_matrix_raw），包含10个基因、6个样本
[工具轨] 正在预处理数据...
[大模型轨] 正在提取摘要...
[系统] 双轨分析完成，正在进行一致性比较...
```

### 步骤来源映射

| track 值 | 前缀标签 |
|----------|----------|
| `init` | [系统] |
| `tool` | [工具轨] |
| `llm` | [大模型轨] |
| `consistency` | [系统] |

### 消息去重

- 当 `status` 发生变化时才追加新消息
- 相同 `status` 的进度更新（如 45% → 46%）不追加消息

### 文件改动

- `src/pages/ChatPage.tsx` - 修改 SSE onmessage 处理逻辑

---

## 2. 追问卡片

### 目标

分析完成后，用户可快速追问系统，展开小型输入框。

### 交互流程

1. 分析完成后，结果卡片底部显示：「还有其他问题吗？」
2. 用户点击「追问」按钮，展开小型输入框
3. 用户输入追问内容，发送到 `/api/chat` 接口
4. 追问结果以普通 assistant 消息追加到会话中

### 组件结构

```
DualTrackResultCard
  └── 追问区域
        ├── "还有其他问题吗？" 文本提示
        ├── [追问] 按钮 (collapsed状态)
        └── 输入框 + 发送按钮 (expanded状态)
```

### 文件改动

- `src/components/DualTrackResultCard.tsx` - 添加追问区域
- `src/pages/ChatPage.tsx` - 处理追问逻辑
